- [ ] **Step 3: 修改 service.go — SendMessage 返回 SendMessageResponse**

修改 `SendMessage` 方法签名从 `(*Conversation, error)` 改为 `(*SendMessageResponse, error)`。

关键改动：
1. 在 AI 分析结果处理时，提取 `status` 和 `metadata` 保存到局部变量
2. 默认 `status = "clarify"`
3. 如果 workflow result 有 status，覆盖默认值

具体修改位置 — 在 `aiResponse` 赋值的同一 block 中：

```go
// 在 var aiResponse 后添加
var aiStatus = "clarify"
var aiMetadata map[string]interface{}

// 在 result 处理块中，提取 status 和 metadata
if status, ok := result["status"].(string); ok {
    aiStatus = status
}
if md, ok := result["metadata"].(map[string]interface{}); ok {
    aiMetadata = md
}

// 如果 status == "confirmed" 且没有 metadata，从 result 顶层提取
if aiStatus == "confirmed" && aiMetadata == nil {
    aiMetadata = make(map[string]interface{})
    for _, key := range []string{"summary", "task_title", "affected_modules", "risk_level", "estimated_complexity"} {
        if v, ok := result[key]; ok {
            aiMetadata[key] = v
        }
    }
}
```

最后返回值改为：
```go
return &SendMessageResponse{
    Conversation: assistantMsg,
    Status:       aiStatus,
    Metadata:     aiMetadata,
}, nil
```

## Task 5: 修复 ExecuteStep — 不要提前标记 COMPLETED

**Files:**

**问题**: ExecuteStep 在 AI activity 运行之前就把 step 标记为 COMPLETED（包含 `time.Sleep` 和 COMPLETED SQL）。正确行为：ExecuteStep 只负责标记 RUNNING（表示 "我正在处理这个步骤"），SaveStepOutput 负责标记 COMPLETED。

**注意**: Task 4 已修改 TaskActivities 添加了 SSEHub 字段。本 task 重写 ExecuteStep 方法时包含 SSE 广播逻辑。

- [ ] **Step 1: 读取 ExecuteStep 确认问题**

读取 `task_activities.go` 中 `ExecuteStep` 方法。确认它先标记 RUNNING 再标记 COMPLETED（且有 `time.Sleep`）。

- [ ] **Step 2: 重写 ExecuteStep — 只标记 RUNNING + SSE 广播**

替换整个 `ExecuteStep` 方法体（Task 4 已添加 `a.sse` 字段和 `task` import）：

```go
func (a *TaskActivities) ExecuteStep(ctx context.Context, input StepInput) (*StepOutput, error) {
    slog.Info("step started", "task_id", input.TaskID, "step", input.StepType, "status", input.TaskStatus)

    // Update task status
    _, err := a.db.Exec(ctx,
        `UPDATE engine.tasks SET status = $1, updated_at = NOW() WHERE id = $2`,
        input.TaskStatus, input.TaskID,
    )
    if err != nil {
        return nil, fmt.Errorf("update task status: %w", err)
    }

    // Mark step as RUNNING (COMPLETED is handled by SaveStepOutput)
    _, err = a.db.Exec(ctx,
        `UPDATE engine.task_steps SET status = 'RUNNING', started_at = NOW() WHERE task_id = $1 AND step_type = $2`,
        input.TaskID, input.StepType,
    )
    if err != nil {
        return nil, fmt.Errorf("mark step running: %w", err)
    }

    // Broadcast SSE event
    if a.sse != nil {
        a.sse.Broadcast(input.TaskID, task.TaskProgressEvent{
            Type:       "STEP_UPDATE",
            TaskID:     input.TaskID,
            Status:     input.TaskStatus,
            StepType:   input.StepType,
            StepStatus: "RUNNING",
        })
    }

    slog.Info("step marked running", "task_id", input.TaskID, "step", input.StepType)
    return &StepOutput{TaskID: input.TaskID, StepType: input.StepType, Status: "RUNNING"}, nil
}
```

删除了：`time.Sleep` 调用、标记 COMPLETED 的 SQL、duration_ms 计算。这些由 SaveStepOutput 处理。

同时检查文件顶部 import：如果 `"time"` 只被 `time.Sleep` 使用，删除该 import（`time.Duration` 也不再需要）。
