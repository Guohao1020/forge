Feature: User Order History

## Overview
Allow authenticated users to view their past orders with pagination support.

## Acceptance Criteria
1. The endpoint GET /api/v1/users/{userId}/orders returns a paginated list of orders.
2. Each order item includes: orderId, status, totalAmount, createdAt.
3. Response is wrapped in Result<Page<OrderVO>>.
4. Returns HTTP 200 with empty page when no orders exist.
5. Returns HTTP 403 when userId does not match the authenticated user.
6. Page size defaults to 20; maximum page size is 100.

## Out of Scope
- Order detail view (covered in separate ticket)
- Order cancellation

## Dependencies
- Requires authentication service (forge-identity)
- Orders stored in orders table in forge-engine DB
